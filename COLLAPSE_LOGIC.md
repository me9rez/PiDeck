# 会话活动轨迹展示逻辑

## 设计目标

会话主线只保留用户问题和 pi 的最终回答；思考、工具调用和回答片段标记统一进入一条可展开的活动轨迹。这样可以在复杂流程里保留真实顺序，同时避免工具调用把回答正文冲散。

## 默认折叠策略

活动轨迹满足以下任一条件时默认展开：

1. 仍有工具调用处于运行中。
2. 工具调用失败，或 bash 等工具结果带有非零退出码。
3. 本轮还没有可显示的最终回答。

其他完成态会话默认折叠，只显示摘要 chip，例如思考段数、工具数量、回答片段数量和警告数量。用户手动展开或收起后，本轮会话保持用户选择，不再被默认规则覆盖。

## 展开态内容

展开后按事件时间顺序展示：

- 思考事件：只有在设置允许显示思考时出现，可继续展开查看全文。
- 工具事件：每次工具调用一行，展示工具名、时间、调用状态和结果状态。
- 回答标记：只显示回答片段预览，用于说明工具/思考与回答之间的相对顺序；完整回答仍在活动轨迹下方展示。

## 工具状态规则

工具调用状态和工具结果状态分开处理：

- `meta.status = running`：工具仍在运行。
- `meta.status = error` 或 `meta.isError = true`：工具调用失败，显示为错误。
- `meta.result.exitCode != 0`：工具调用成功，但结果需要留意，显示为警告。
- 其他完成工具：显示为完成。

bash 的非零退出码不等同于 RPC 工具失败，因为它可能只是命令自身的业务结果。

- `src/renderer/src/components/app/AppParts.tsx`

  - `ActivityEventRow`：思考、工具、回答标记的统一事件行。
- `src/renderer/src/styles.css`
  - `.run-activity*` 和 `.activity-event*`：活动轨迹视觉样式。
- `src/renderer/src/i18n.ts`
  - `activity.*`：活动轨迹文案。
